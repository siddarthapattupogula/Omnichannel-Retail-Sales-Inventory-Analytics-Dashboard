# Omnichannel-Retail-Sales-Inventory-Analytics-Dashboard
Project Overview

This project focuses on analyzing retail sales data to identify key business insights. The objective is to build an end-to-end data analytics pipeline that cleans raw sales data, performs SQL analysis, and creates an interactive Power BI dashboard to visualize sales trends, product performance, and geographic distribution.

👉Business Problem

Retail businesses operating across multiple locations often face challenges in consolidating sales data from different channels. Without centralized analytics, it becomes difficult to monitor sales performance, track inventory movement, and identify high-performing products.

This project addresses these challenges by developing a centralized analytics dashboard that transforms raw transaction data into meaningful business insights.

🔍Project Objectives

1.Analyze retail sales data to identify important revenue trends
2.Calculate key performance indicators such as Total Revenue and Average Revenue per Order
3.Identify best-selling products and product categories
4.Analyze sales performance across different cities
5.Identify peak purchasing hours and seasonal trends
6.Build an interactive Power BI dashboard for business decision making

👉Technology Stack
1.Python (Pandas) – Data Cleaning and Processing
2.Excel – Initial Data Exploration
3.SQL (MySQL / PostgreSQL) – Data Querying and Analysis
4.Power BI – Data Visualization and Dashboard Development
5.GitHub – Version Control and Project Documentation

📊Project Workflow

📅Week 1   –   Data Cleaning and Preparation
1.Imported raw retail dataset into Python and Excel
2.Removed duplicate records and handled missing values
3.Standardized date and time formats
4.Verified data types and prepared clean dataset for analysis

📅Week 2   –   SQL Database and Data Analysis
1.Imported cleaned dataset into SQL database
2.Performed SQL queries using GROUP BY, ORDER BY, SUM, and AVG
3.Calculated business metrics including total revenue, best-selling products, and sales distribution by city

📅Week 3   –  Dashboard Development
1.Connected dataset to Power BI
2.Created interactive visualizations including sales trends, product performance, and geographic analysis
3.Implemented slicers and filters for dynamic data exploration

📅 Week 4 – Insights & Final Submission

In the final phase of the project, the focus was on analyzing the Power BI dashboard to extract meaningful business insights and providing actionable recommendations.

🔍 Key Activities
Interpreted dashboard visualizations to identify sales patterns and trends
Extracted key insights such as peak sales periods, top-performing products, and high-revenue cities
Analyzed seasonal trends and customer purchasing behavior
📊 Key Insights
Sales showed clear seasonal trends across months
A few products contributed to the majority of total revenue
Certain cities generated significantly higher sales
Peak purchasing activity occurred during specific hours and weekdays
💡 Business Recommendations
Increase inventory for high-demand products
Focus marketing efforts on high-performing regions
Run targeted promotions during peak sales periods
Optimize staffing during high-traffic hours

Completed Power BI dashboard
Documented key insights and business strategies
Prepared final project report
